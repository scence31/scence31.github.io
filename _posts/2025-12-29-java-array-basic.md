## 📌 1. 배열의 의미와 선언
데이터가 많아지면 변수를 일일이 만들기 힘들어지죠. 이때 같은 타입의 값들을 묶어서 관리하는 게 **배열**입니다.

* **표현법**: `int[] arr = new int[3];`
* **메모리 흐름**:
    1. **배열 선언**: Stack 영역에 `arr`이라는 빈 상자가 만들어집니다.
    2. **배열 할당**: `new` 구문을 통해 Heap 영역에 방(공간)이 3개 만들어집니다.
    3. **연결**: Heap에 생긴 방의 주소값이 Stack의 `arr` 상자에 저장됩니다.

> **💡 핵심**: 인덱스는 항상 **0**부터 시작한다는 점! `arr[3]`을 호출하면 `ArrayIndexOutOfBoundsException` 에러를 만나게 되니 주의해야 해요.

## 📌 2. 배열을 왜 쓸까?
단순히 "편해서"라기보다 효율성 때문입니다. 
1. 변수가 많아질 때 일일이 이름을 지어줄 필요가 없어요.
2. `for`문과 연계해서 반복 처리가 가능합니다. (수학적 계산이 가능해짐)
* **단점**: 처음에 정한 크기를 중간에 바꿀 수 없습니다. (바꾸려면 `new`로 새로 만들어야 함)

```java
// 가장 많이 쓰는 패턴
for(int i = 0; i < arr.length; i++) {
    System.out.println(arr[i]);
}
