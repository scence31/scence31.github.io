## 컬렉션 프레임워크란?
대량의 데이터를 쉽고 효과적으로 처리할 수 있도록 표준화된 방법을 제공하는 클래스의 집합

* **배열의 한계**: 크기가 고정되어 있어 데이터를 추가/삭제하기 불편함
* **컬렉션의 장점**: 데이터의 개수에 따라 크기가 자동으로 조절되며, 다양한 자료구조를 미리 만들어 두어 가져다 쓰기만 하면 됨

---

## 핵심 인터페이스 3대장
컬렉션은 크게 `List`, `Set`, `Map` 세 가지 인터페이스로 나뉨



### 1. List (순서 O, 중복 O)
- 가장 배열과 비슷하게 쓰이며, 데이터를 일렬로 나열한 구조
- **종류**: `ArrayList` 등
- **특징**: 인덱스로 관리하기 때문에 특정 위치의 데이터를 찾기 쉬움

### 2. Set (순서 X, 중복 X)
- 집합 개념으로 주머니 속에 구슬을 넣어두는 것과 같아서 순서가 없고 중복을 허용하지 않음
- **종류**: `HashSet`, `TreeSet` 등
- **특징**: 중복 데이터를 자동으로 걸러낼 때 유용

### 3. Map (Key-Value 세트)
- 사전과 같은 구조로 단어(Key)와 뜻(Value)이 한 세트로 묶임
- **종류**: `HashMap`, `Properties` 등
- **특징**: Key는 중복될 수 없고, Value는 중복 가능함. (key 값이 중복되면 나중에 쓴 key 값으로 덮어씌워짐)

## 제네릭(Generic) 설정
컬렉션을 사용할 때 `<T>`와 같은 모양을 보게 되는데, 이는 상자에 담을 타입을 미리 정해주는 것임

```java
// String만 담을 수 있는 ArrayList 생성
List<String> list = new ArrayList<>(); 
list.add("Java");
// list.add(10); // 에러 발생! (타입 안정성 확보)
```

* `add()`: 데이터 추가
* `remove()`: 데이터 삭제
* `size()`: 데이터 개수 확인(== length)
* `get()`: 데이터 꺼내기(List 전용)
* `clear()`: 값 비우기
